# Data Quality Scripts

This document covers the correctional and quality-gate scripts that accompany the
main data-generation pipeline. They are standalone Python scripts, each addressing
a specific quality dimension of the dataset. All use the Anthropic API
(`ANTHROPIC_API_KEY`) and are safe to re-run at any time with `--dry_run` before
committing changes.

---

## Overview

The generation pipeline intentionally separates **generation** from **quality
control**: each Phase generates data as fast as possible, and the quality scripts
below are then run as a second pass to audit and cull problematic rows. This keeps
the generation scripts simple and makes quality decisions explicit and reviewable.

```
Phase 1a  ──►  fix_scratch_realism.py     (cull low-realism high-risk rows)
               fix_scratch_low_risk.py    (cull accidentally-risky low-risk rows)

Phase 1b/2g ──►  (quality gate built into CAMEL/PAIR generation loop)

Phase 2a-2d ──►  fix_augmented_labels.py  (cull signal-diluted augmented rows)
                 fix_softened_rows.py      (cull over-softened rows)

Post-clean  ──►  reannotate_augmented.py  (re-annotate after artifact removal)
```

---

## `fix_scratch_realism.py`

**Purpose:** Realism gate for Phase 1a (`generate_scratch.py`) high-risk rows.

CAMEL and PAIR both apply a Claude Sonnet realism filter (score ≥ 7) at generation
time. Phase 1a uses bulk LLM generation without an equivalent gate, so some rows
may be overly clinical, formulaic, or AI-sounding. This script runs every high-risk
row through Claude Sonnet and removes those scoring below the threshold.

Low-risk rows are handled separately by `fix_scratch_low_risk.py`.

```bash
# Back up first
cp datasets/generated_scratch.csv datasets/generated_scratch.csv.bak

# Dry run (see what would be removed, no writes)
python project/scripts/fix_scratch_realism.py \
    --input datasets/generated_scratch.csv \
    --dry_run

# Live run (default threshold=5, lower than CAMEL/PAIR's 7 — adjust upward for stricter culling)
python project/scripts/fix_scratch_realism.py \
    --input datasets/generated_scratch.csv \
    --threshold 5
```

**Env vars required:** `ANTHROPIC_API_KEY`

## `fix_scratch_low_risk.py`

**Purpose:** Quality gate for Phase 1a low-risk rows (accidental signal contamination).

Low-risk rows (`high_risk_any=0`) generated by Phase 1a had no external validation
at generation time. Some may accidentally contain real distress signals — particularly
subtle ones like `testing` or `third_party_concern`. This script runs every low-risk
row through Claude Sonnet and removes any that score above the threshold on any signal.

```bash
cp datasets/generated_scratch.csv datasets/generated_scratch.csv.bak

python project/scripts/fix_scratch_low_risk.py \
    --input datasets/generated_scratch.csv \
    --threshold 6 \
    [--dry_run]
```

**Env vars required:** `ANTHROPIC_API_KEY`

**Note:** Run this *after* `fix_scratch_realism.py` — both operate on the same file
and either order is safe, but running realism first reduces the number of rows that
the low-risk scan needs to process.

## `fix_augmented_labels.py`

**Purpose:** Signal-survival check for Phase 2a/2b augmented rows.

`augment_seed.py` Phases 2a (language rewrite) and 2b (persona swap / ESConv swap)
inherit signal labels from the parent seed row without verifying that the signal
survived the transformation. A French rewrite or a persona swap to a very different
age/register can dilute or erase the original signal while the label remains
`high_risk_any=1`. This is a silent quality drag: the row trains the classifier on
weak or absent signal expression.

This script re-scores every high-risk augmented row with source in
`{lang_rewrite, persona_swap, esconv_swapped}` using Claude Sonnet and removes rows
where no active signal scores above the threshold.

```bash
cp datasets/augmented.csv datasets/augmented.csv.bak

python project/scripts/fix_augmented_labels.py \
    --input datasets/augmented.csv \
    [--threshold 5] \
    [--dry_run]
```

**Env vars required:** `ANTHROPIC_API_KEY`

**Note:** Run *after* `fix_softened_rows.py` (separate concern, separate pass — either order is safe).

---

## `fix_softened_rows.py`

**Purpose:** Quality gate for Phase 2d signal-softening rows.

`augment_seed.py` Phase 2d (signal softening) had no external quality gate at
generation time. The softening LLM may over-soften conversations, producing rows
labeled `high_risk_any=1` where the signal has effectively disappeared. These are
the most dangerous mislabeled rows in the dataset: a high-risk label on content that
a careful reader would find low-risk.

This script re-scores every `source=signal_soften` row using Claude Sonnet and
removes those where no active signal scores above the threshold.

```bash
cp datasets/augmented.csv datasets/augmented.csv.bak

python project/scripts/fix_softened_rows.py \
    --input datasets/augmented.csv \
    [--threshold 5] \
    [--dry_run]
```

**Env vars required:** `ANTHROPIC_API_KEY`

## `reannotate_augmented.py`

**Purpose:** Re-annotate signal labels after artifact removal.

The dataset cleaning step (`clean_augmented.py`) strips LLM wrapper artifacts (e.g.
`"Sure, here is..."` preambles, `"[END]"` suffixes) from augmented rows and records
which rows were cleaned via a `needs_reannotation` column. This script re-annotates
those rows using Claude Sonnet so that signal labels reflect the **cleaned text**
rather than the original contaminated version. Untouched rows retain their existing
annotations.

After running, the `needs_reannotation` and `artifact_type` tracking columns are
stripped from the output to restore the standard `augmented.csv` column schema.

Checkpointing is built in: progress is saved every `CHECKPOINT_INTERVAL` rows so
the script can be safely interrupted and resumed.

```bash
# In-place (reads entirely into memory before writing — safe)
python project/scripts/reannotate_augmented.py \
    --input  datasets/augmented.csv \
    --output datasets/augmented.csv

# Or write to a separate file for safety
python project/scripts/reannotate_augmented.py \
    --input  datasets/augmented.csv \
    --output datasets/augmented_reannotated.csv
```

**Env vars required:** `ANTHROPIC_API_KEY`

## Recommended run order

When regenerating the full dataset from scratch, run the quality scripts in this order:

```bash
# 1. After Phase 1a (generate_scratch.py)
python project/scripts/fix_scratch_realism.py  --input datasets/generated_scratch.csv
python project/scripts/fix_scratch_low_risk.py --input datasets/generated_scratch.csv

# 2. After Phase 2a-2d (augment_seed.py)
python project/scripts/fix_softened_rows.py    --input datasets/augmented.csv
python project/scripts/fix_augmented_labels.py --input datasets/augmented.csv

# 3. After clean_augmented.py (artifact removal)
python project/scripts/reannotate_augmented.py \
    --input datasets/augmented.csv --output datasets/augmented.csv
```

Phases 1b/2g (CAMEL, PAIR) have their quality gates built into the generation loop
and do not require a separate pass.
